# 法智汇全链路一体化SaaS产品实施规划

## Why
需求文档《网推律所全链路一体化SaaS产品需求文档V1.0》定义了完整的8大核心模块功能，但现有系统仅实现了部分基础功能。需要按照需求文档逐步补全所有功能，确保"一个功能一个功能实现，全部实现后再检查流程是否闭环"。

## What Changes
本次规划覆盖8大核心模块的完整实现：

- **模块1：全域投放与获客营销系统** - 新增广告账户管理、投放计划、转化归因、素材管理、AI内容生成、合规预审、账号矩阵
- **模块2：公私域连接器与SCRM私域运营系统** - 新增活码管理、渠道追踪、客户标签、企微侧边栏、私域触达、聊天存档
- **模块3：线索中台与谈案转化CRM** - 补全线索分配引擎、公海池、邀约岗/谈案岗专属管理、谈案SOP、AI合规辅助、资产交接
- **模块4：标准化案件办案管理系统** - 补全办案SOP模板、任务协同、节点预警、证据卷宗、法律检索、文书生成、OA审批
- **模块5：全节点AI合规风控体系** - 新增获客合规、谈案质检、签约合规、办案合规、结案合规、财务合规、客诉闭环
- **模块6：财务分润与收支管理系统** - 补全收费管理、分润引擎、成本核算、退费管理、财务报表
- **模块7：C端客户服务与口碑运营体系** - 补全案件推送、AI答疑、线上服务大厅、服务评价
- **模块8：全链路经营数据决策中台** - 新增转化漏斗、绩效看板、办案效能、财务看板、合规监控、自定义报表

## Impact
- **Affected specs**: 所有8大功能模块
- **Affected code**: 
  - 前端：新增约30+页面组件
  - 后端：新增约20+模块/实体
  - 数据库：新增约40+实体表

## 实施策略

### 阶段划分
按照业务流程优先级，分为以下阶段：

**阶段一：核心业务闭环（优先级最高）**
- 补全线索中台与谈案转化CRM的关键功能
- 补全案件办案管理的关键功能
- 补全财务分润系统的关键功能
- 目标：实现"线索→签约→办案→收费"核心闭环

**阶段二：合规风控体系**
- 实现全节点AI合规风控体系
- 目标：覆盖全链路合规管控

**阶段三：营销获客体系**
- 实现全域投放与获客营销系统
- 实现公私域连接器与SCRM私域运营系统
- 目标：打通前端获客链路

**阶段四：客户服务与数据中台**
- 补全C端客户服务功能（7.1客户专属端口增强、7.2案件进度推送、7.3 AI答疑增强、7.4线上服务大厅、7.5服务评价沉淀）
- 实现全链路经营数据决策中台（8.1转化漏斗看板、8.2销售绩效看板、8.3办案效能看板、8.4财务经营看板、8.5合规风险监控、8.6自定义报表导出）
- 前置任务F1：补全 Phase 1-3 前端路由与菜单注册（19个页面）
- 目标：提升客户体验与决策支撑

### 阶段四新增实体
- `CasePushNotification` - 案件进度推送记录
- `ClientConsultation` - 客户咨询记录（含转人工工单）
- `ServiceRating` - 服务评价（含口碑沉淀）
- `ReportTemplate` - 自定义报表模板（含订阅配置）
- `ReportExportLog` - 报表导出日志

### 实施原则
1. **一个功能一个功能实现**：每个功能点独立开发、测试、验收
2. **数据流向闭环验证**：每个功能完成后验证上下游数据衔接
3. **模块级集成测试**：每个模块完成后进行模块内闭环验证
4. **系统级流程验证**：全部模块完成后进行全链路闭环验证
